## Project Overview

This is a WordPress e-commerce theme built on the Sage 11 starter theme framework, integrated with WooCommerce, Livewire, and custom payment processing via Mollie. The theme uses Laravel Blade templating, Tailwind CSS v4, and Vite for modern front-end development.

## Architecture

### Framework Stack
- **Sage 11**: WordPress starter theme with Laravel components via Roots Acorn
- **Acorn**: Laravel application container for WordPress (provides IoC, service providers, routing)
- **WooCommerce**: E-commerce functionality with custom Livewire components replacing standard WC templates
- **Livewire 3.6**: Full-page components for cart, checkout, account, favorites, product categories
- **Tailwind CSS v4**: Utility-first CSS framework with `@tailwindcss/vite` plugin
- **Vite**: Build tool with HMR and asset bundling

### Service Providers
Service providers are registered in `functions.php`:
- `ThemeServiceProvider`: Core theme functionality
- `WooCommerceServiceProvider`: WooCommerce customizations
- `MollieServiceProvider`: Mollie payment gateway integration
- `InvoiceServiceProvider`: PDF invoice generation with WooCommerce admin integration
- `TNTSearchProvider`: Full-text search engine with real-time index updates

### Routing
Custom routes are defined in `routes/web.php` using Laravel routing:
- `/afrekenen/` - Checkout (Livewire component)
- `/winkelwagen/` - Cart (Livewire component)
- `/favorieten/` - Favorites (Livewire component)
- `/account/` - My Account (Livewire component)
- `/zoeken/` - Site search (Livewire component)
- `/afrekenen/bedankt/{order_id}/` - Thank you page with time-limited access
- `/factuur/{order_id}/preview/` - Invoice preview (admin only, nonce-protected)
- `/factuur/{order_id}/download/` - Invoice download (admin only, nonce-protected)

### Livewire Integration
The theme uses Livewire 3.6 for interactive components. Major Livewire components are located in `app/Livewire/`:
- `Checkout.php`: Full checkout flow
- `DeliveryOptions.php`: Shipping/delivery selection
- `Cart.php`: Shopping cart
- `Favorites.php`: Wishlist functionality
- `MyAccount.php`: Customer account management
- `SiteSearch.php`: Full-page search with multi-type results
- `HeaderSearch.php`: Inline header search component
- `RecentlyViewed.php`: Recently viewed products tracking

### WooCommerce Customization
- Custom template hooks in `app/wc-template-hooks.php`
- Template overrides in `resources/views/woocommerce/`
- Custom payment gateways in `app/Woocommerce/Gateways/`
- Integration handled by `generoi/sage-woocommerce` package

### View Components
Blade components in `app/View/Components/` with corresponding templates in `resources/views/components/`:
- Form elements: `Button`, `Checkbox`, `Radio`, `Select`, `InputText`, `TextArea`, `FieldGroup`
- UI elements: `Modal`, `SlideOut`, `Accordion`, `Carousel`, `Pill`
- Commerce: `Product`, `CheckoutStep`

### Third-Party Services
Configuration in `config/services.php` and dedicated config files:
- **Mollie**: Payment processing (`MollieService`)
- **Mailchimp**: Newsletter subscriptions
- **Lightspeed Retail API**: POS integration
- **Kadaster**: Address validation service (`KadasterService`)
- **reCAPTCHA**: Form protection
- **DomPDF**: PDF generation for invoices (`InvoiceService`) - configured in `config/dompdf.php`
- **TNTSearch**: Full-text search engine (`TNTSearchService`) - configured in `config/search.php`

### Invoice Generation
The theme includes a complete invoice generation system using DomPDF:
- **Service**: `InvoiceService` provides PDF generation from WooCommerce orders
- **Controller**: `InvoiceController` handles preview/download routes with nonce security
- **Templates**: `resources/views/pdf/invoice.blade.php` for invoice layout
- **WooCommerce Integration**: Meta boxes for custom invoice numbers/dates, action buttons in order list
- **Access Control**: Admin-only access with nonce verification
- **Invoice Numbers**: Format `DB-{order_number}` (customizable per order)
- **Status Filter**: Only generates for processing/completed/on-hold orders

### Site Search
The theme uses TNTSearch for fast full-text search across multiple content types:
- **Search Service**: `TNTSearchService` handles multi-type search (products, categories, tags, brands)
- **Configuration**: `config/search.php` with fuzzy matching, synonyms, and index settings
- **Indexes**: Separate indexes for products, categories, tags, and brands stored in `wp-content/tntsearch/`
- **Real-time Updates**: Automatic index updates via WordPress hooks when content is created/updated
- **Custom Tokenizer**: `PrefixTokenizer` for prefix matching and autocomplete support
- **Analytics**: `SearchAnalyticsService` for tracking search queries and results
- **Ranking**: Custom relevance ranking prioritizing title matches and prefix matches
- **Synonym Support**: Configurable synonym groups (e.g., "whisky" ↔ "whiskey")
- **Console Commands**: WP-CLI commands for index management in `app/Console/Commands/Search/`

#### Search Console Commands
```bash
# Rebuild all search indexes
wp acorn search:index-all

# Rebuild individual indexes
wp acorn search:index-products
wp acorn search:index-categories
wp acorn search:index-tags
wp acorn search:index-brands

# Create analytics table
wp acorn search:create-analytics-table
```

## Development Workflow

### Initial Setup
```bash
# Install PHP dependencies
composer install

# Install Node dependencies
npm install

# Build assets for development
npm run dev
```

### Development Commands
```bash
# Start Vite dev server with HMR
npm run dev

# Build for production
npm run build

# Format PHP code with Laravel Pint
./vendor/bin/pint

# Generate translation POT file
npm run translate:pot

# Update PO files from POT
npm run translate:update

# Compile translations (MO and JSON)
npm run translate:compile

# Rebuild all search indexes
wp acorn search:index-all

# Create search analytics table
wp acorn search:create-analytics-table
```

## Key Technical Details

### Autoloading
PSR-4 autoloading is configured in `composer.json`:
- Namespace `App\` maps to `app/` directory

### ACF Integration
Uses `log1x/acf-composer` for programmatic ACF field group creation. Configuration in `config/acf.php`.

### Laravel Features Available
Via Roots Acorn:
- Service Container & Dependency Injection
- Service Providers
- Routing (see `routes/web.php`)
- Blade Templating
- View Composers
- Facades
- Collections
- Mail (via `roots/acorn-mail`)

### Environment Variables
The theme expects a `.env` file in the theme root with:
- `MOLLIE_API_KEY`, `MOLLIE_PROFILE_ID`, `MOLLIE_TEST_MODE`
- `MAILCHIMP_API_KEY`, `MAILCHIMP_LIST_ID`
- `RECAPTCHA_SITE_KEY`, `RECAPTCHA_SECRET_KEY`
- `SEARCH_FUZZY_ENABLED`, `SEARCH_FUZZY_PREFIX`, `SEARCH_FUZZY_MAX_EXPANSIONS`, `SEARCH_FUZZY_DISTANCE` (optional)
- Database and mail service credentials (if using Acorn's database features)

### Theme Support
Configured in `app/setup.php`:
- Block templates are disabled (not a block theme)
- Responsive embeds, post thumbnails, HTML5 markup
- Navigation menus (primary_navigation)
- Sidebars: `sidebar-primary`, `sidebar-footer`

### Custom WooCommerce Thank You Page
`routes/web.php` implements a time-limited thank you page that expires after 5 minutes unless the user is an admin/shop manager.

## Common Tasks

### Adding a New Livewire Component
1. Create component in `app/Livewire/ComponentName.php`
2. Create view in `resources/views/livewire/component-name.blade.php`
3. If full-page component, add route in `routes/web.php`

### Adding a New Blade Component
1. Create component class in `app/View/Components/ComponentName.php`
2. Create template in `resources/views/components/component-name.blade.php`
3. Use in templates: `<x-component-name />`

### Adding a New Composer 
1. Use command wp acorn make:composer ComposerName
2. Never fetch data inside a blade view, always use a composer to pass it down
3. Composers work downstream only, you cannot use them as a class if you need the functionality somewhere else, create service for it
4. Composers are auto registered if a variable is not defined it means there is an error somewhere

### Adding a New Service
1. Create service class in `app/Services/ServiceName.php`
2. Register in service provider if needed for DI
3. Access via `app(ServiceName::class)` or constructor injection

### Customizing WooCommerce Templates
1. Create override in `resources/views/woocommerce/` matching WC structure
2. Use Blade syntax instead of PHP templates
3. Reference `app/wc-template-hooks.php` for available hooks to remove/modify

### Adding Translation Strings
1. Use `__('String', 'sage')` in PHP/Blade
2. Use `_e('String', 'sage')` for direct echo
3. Run `npm run translate` to update POT and PO files
4. Compile with `npm run translate:compile`

### Working with Search
1. Search indexes are automatically updated when content changes (products, categories, tags, brands)
2. To manually rebuild indexes: `wp acorn search:index-all`
3. Configure search behavior in `config/search.php`
4. Add synonyms to improve search relevance
5. Search analytics are logged to track popular queries and result counts

### Generating Invoices
1. Invoices are automatically available for processing/completed/on-hold orders
2. Access via WooCommerce admin order page (preview/download buttons)
3. Customize invoice number/date using the meta box in order edit screen
4. PDF templates are in `resources/views/pdf/`
5. Use `InvoiceService` for programmatic access to invoice generation

### Coding style / instructions 
1. Always check if there is a matching blade component when writing html 
2. If there is no blade component. Add one, never start duplicating large or complex sections of html 
3. Create blade components using wp acorn make:component ComponentName. 
4. Don't use the view class but rather the anonymous variant with @props in the blade view
5. If the component is complex e.g. needs logic for data ask first
6. Use the @svg directive when adding icons always, there is no <x-icon component.
