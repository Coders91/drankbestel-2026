## Project Overview

This is a WordPress e-commerce theme built on the Sage 11 starter theme framework, integrated with WooCommerce, Livewire, and custom payment processing via Mollie. The theme uses Laravel Blade templating, Tailwind CSS v4, and Vite for modern front-end development.

## Architecture

### Framework Stack
- **Sage 11**: WordPress starter theme with Laravel components via Roots Acorn
- **Acorn**: Laravel application container for WordPress (provides IoC, service providers, routing)
- **WooCommerce**: E-commerce functionality with custom Livewire components replacing standard WC templates
- **Livewire 3.6**: Full-page components for cart, checkout, account, favorites, product categories
- **Tailwind CSS v4**: Utility-first CSS framework with `@tailwindcss/vite` plugin
- **Vite**: Build tool with HMR and asset bundling

### Directory Structure
```

├── Http/Controllers/     # Laravel-style controllers
├── Livewire/            # Livewire full-page components (checkout, cart, account, etc.)

├── Providers/           # Service providers (Theme, WooCommerce, Mollie)
├── Services/            # Business logic services (MollieService, KadasterService)
├── View/
│   ├── Components/      # Blade components (Button, Modal, Accordion, etc.)
│   └── Composers/       # View composers for passing data to views
├── Woocommerce/
│   └── Gateways/        # Custom WooCommerce payment gateways
├── filters.php          # WordPress filters
├── setup.php            # Theme setup and WordPress hooks
└── wc-template-hooks.php # WooCommerce hook customizations

config/                  # Laravel-style configuration files
├── acf.php             # ACF Composer configuration
├── livewire.php        # Livewire settings
├── services.php        # Third-party service credentials (Mollie, Mailchimp, etc.)
└── ...

resources/
├── css/                # Tailwind CSS source files
├── js/                 # JavaScript entrypoints
└── views/              # Blade templates
    ├── components/     # Blade component templates
    ├── sections/       # Header, footer, sidebar
    ├── woocommerce/    # WooCommerce template overrides
    └── ...

└── web.php             # Laravel-style routes (custom WC pages: checkout, cart, account)

public/build/           # Compiled assets (generated by Vite)
```

### Service Providers
Service providers are registered in `functions.php`:
- `ThemeServiceProvider`: Core theme functionality
- `WooCommerceServiceProvider`: WooCommerce customizations
- `MollieServiceProvider`: Mollie payment gateway integration

### Routing
Custom routes are defined in `routes/web.php` using Laravel routing:
- `/afrekenen/` - Checkout (Livewire component)
- `/winkelwagen/` - Cart (Livewire component)
- `/favorieten/` - Favorites (Livewire component)
- `/account/` - My Account (Livewire component)
- `/afrekenen/bedankt/{order_id}/` - Thank you page with time-limited access

### Livewire Integration
The theme uses Livewire 3.6 for interactive components. Major Livewire components are located in `app/Livewire/`:
- `Checkout.php`: Full checkout flow
- `DeliveryOptions.php`: Shipping/delivery selection
- `Cart.php`: Shopping cart
- `Favorites.php`: Wishlist functionality
- `MyAccount.php`: Customer account management

### WooCommerce Customization
- Custom template hooks in `app/wc-template-hooks.php`
- Template overrides in `resources/views/woocommerce/`
- Custom payment gateways in `app/Woocommerce/Gateways/`
- Integration handled by `generoi/sage-woocommerce` package

### View Components
Blade components in `app/View/Components/` with corresponding templates in `resources/views/components/`:
- Form elements: `Button`, `Checkbox`, `Radio`, `Select`, `InputText`, `TextArea`, `FieldGroup`
- UI elements: `Modal`, `SlideOut`, `Accordion`, `Carousel`, `Pill`
- Commerce: `Product`, `CheckoutStep`

### Third-Party Services
Configuration in `config/services.php`:
- **Mollie**: Payment processing (`MollieService`)
- **Mailchimp**: Newsletter subscriptions
- **Lightspeed Retail API**: POS integration
- **Kadaster**: Address validation service (`KadasterService`)
- **reCAPTCHA**: Form protection

## Development Workflow

### Initial Setup
```bash
# Install PHP dependencies
composer install

# Install Node dependencies
npm install

# Build assets for development
npm run dev
```

### Development Commands
```bash
# Start Vite dev server with HMR
npm run dev

# Build for production
npm run build

# Format PHP code with Laravel Pint
./vendor/bin/pint

# Generate translation POT file
npm run translate:pot

# Update PO files from POT
npm run translate:update

# Compile translations (MO and JSON)
npm run translate:compile
```

### Asset Configuration
Vite configuration (`vite.config.js`) uses different base paths for dev vs production:
- **Development**: `/app/themes/drankbestel-new/public/build/`
- **Production**: `/wp-content/themes/drankbestel-new/public/build/`

Entry points:
- `resources/css/app.css` - Main styles
- `resources/js/app.js` - Main scripts
- `resources/css/editor.css` - Block editor styles
- `resources/js/editor.js` - Block editor scripts

### Path Aliases
Configured in `vite.config.js`:
- `@scripts` → `/resources/js`
- `@styles` → `/resources/css`
- `@fonts` → `/resources/fonts`
- `@images` → `/resources/images`

## Key Technical Details

### Autoloading
PSR-4 autoloading is configured in `composer.json`:
- Namespace `App\` maps to `app/` directory

### ACF Integration
Uses `log1x/acf-composer` for programmatic ACF field group creation. Configuration in `config/acf.php`.

### Laravel Features Available
Via Roots Acorn:
- Service Container & Dependency Injection
- Service Providers
- Routing (see `routes/web.php`)
- Blade Templating
- View Composers
- Facades
- Collections
- Mail (via `roots/acorn-mail`)

### Environment Variables
The theme expects a `.env` file in the theme root with:
- `MOLLIE_API_KEY`, `MOLLIE_PROFILE_ID`, `MOLLIE_TEST_MODE`
- `MAILCHIMP_API_KEY`, `MAILCHIMP_LIST_ID`
- `RECAPTCHA_SITE_KEY`, `RECAPTCHA_SECRET_KEY`
- Database and mail service credentials (if using Acorn's database features)

### Theme Support
Configured in `app/setup.php`:
- Block templates are disabled (not a block theme)
- Responsive embeds, post thumbnails, HTML5 markup
- Navigation menus (primary_navigation)
- Sidebars: `sidebar-primary`, `sidebar-footer`

### Custom WooCommerce Thank You Page
`routes/web.php` implements a time-limited thank you page that expires after 5 minutes unless the user is an admin/shop manager.

## Common Tasks

### Adding a New Livewire Component
1. Create component in `app/Livewire/ComponentName.php`
2. Create view in `resources/views/livewire/component-name.blade.php`
3. If full-page component, add route in `routes/web.php`

### Adding a New Blade Component
1. Create component class in `app/View/Components/ComponentName.php`
2. Create template in `resources/views/components/component-name.blade.php`
3. Use in templates: `<x-component-name />`

### Adding a New Service
1. Create service class in `app/Services/ServiceName.php`
2. Register in service provider if needed for DI
3. Access via `app(ServiceName::class)` or constructor injection

### Customizing WooCommerce Templates
1. Create override in `resources/views/woocommerce/` matching WC structure
2. Use Blade syntax instead of PHP templates
3. Reference `app/wc-template-hooks.php` for available hooks to remove/modify

### Adding Translation Strings
1. Use `__('String', 'sage')` in PHP/Blade
2. Use `_e('String', 'sage')` for direct echo
3. Run `npm run translate` to update POT and PO files
4. Compile with `npm run translate:compile`
